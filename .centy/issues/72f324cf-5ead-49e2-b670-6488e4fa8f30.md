# Implement clippy::too_many_lines lint

## Overview

Implement the [`clippy::too_many_lines`](https://rust-lang.github.io/rust-clippy/master/index.html#too_many_lines) pedantic lint as a Dylint lint.

## Tasks

- Study the clippy reference implementation for `clippy::too_many_lines`
- Implement the lint in a new Dylint lint crate (or extend an existing one)
- Write UI tests covering the lint's detection and allowed cases
- Ensure the lint passes `cargo clippy`, `cargo fmt`, and `cargo test`

## Notes

- Lint group: **pedantic**
- Clippy reference: <https://rust-lang.github.io/rust-clippy/master/index.html#too_many_lines>

## Acceptance Criteria

- Lint correctly detects the pattern described by `clippy::too_many_lines`
- False positives are handled (lint does not fire on allowed code)
- UI test `.stderr` file matches actual output

# Implement NO_EXPECT lint

## Overview

Implement a `NO_EXPECT` Dylint lint that forbids the use of `.expect()` on `Option` and `Result` types.

## Tasks

- Implement the lint in a new Dylint lint crate (or extend an existing one)
- Detect calls to `.expect()` on `Option<T>` and `Result<T, E>`
- Write UI tests covering detection and allowed cases
- Ensure the lint passes `cargo clippy`, `cargo fmt`, and `cargo test`

## Notes

- This lint encourages explicit error handling instead of panicking with a message on `None`/`Err`
- Consider allowing `expect()` in test code (`#[cfg(test)]`) or via `#[allow(...)]`
- Related clippy lints: `clippy::expect_used`

## Acceptance Criteria

- Lint correctly detects `.expect()` calls on `Option` and `Result`
- False positives are handled (lint does not fire on allowed code)
- UI test `.stderr` file matches actual output

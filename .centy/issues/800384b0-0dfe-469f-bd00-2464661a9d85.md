# Implement NO_UNWRAP lint

## Overview

Implement a `NO_UNWRAP` Dylint lint that forbids the use of `.unwrap()` on `Option` and `Result` types.

## Tasks

- Implement the lint in a new Dylint lint crate (or extend an existing one)
- Detect calls to `.unwrap()` on `Option<T>` and `Result<T, E>`
- Write UI tests covering detection and allowed cases
- Ensure the lint passes `cargo clippy`, `cargo fmt`, and `cargo test`

## Notes

- This lint encourages explicit error handling instead of panicking on `None`/`Err`
- Consider allowing `unwrap()` in test code (`#[cfg(test)]`) or via `#[allow(...)]`
- Related clippy lints: `clippy::unwrap_used`

## Acceptance Criteria

- Lint correctly detects `.unwrap()` calls on `Option` and `Result`
- False positives are handled (lint does not fire on allowed code)
- UI test `.stderr` file matches actual output
